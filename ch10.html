Chapter 10 : How To Evaluate Performance

Key concepts in this chapter:

predict()

confusion metrics

accuracy

error rates

Kappa statistic Sensitivity and specificity Precision and recall F-measure

ROC curves

Key Point 1:

In evaluating performance of the data, probability analysis is important, especially in using confusion metrics to measure performance. The cross table has been used again. Among them, the Kappa statistic, sensitivity and specificity, precision and recall, and the F measure are frequently used. The concept of identifying mutually exclusive events in the model is also used in evaluation.

It is easier to do the Kappa statistic when draw the data in table manner.

Key Point 2:

In real world problems, the cross-validation concepts is quite useful. In documents published by Microsoft on Axure’s evaluation of model’s performance, Cross-validation is named one of the most important tool to evaluate the model’s performance. Cross-validation can be used to retain and reevaluate the model. Also, the fact that it trains 10 folds of data, reduce computation calculation and is more efficient. It will be interesting to use the leave-one-out- method: use only one fold for each of the data’s example.

Every evaluation model has its trade-offs, it should consider cautiously them and more practice will have the process.
