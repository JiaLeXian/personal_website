Chapter 3 : K-NN Algorithm

The powerful K-NN algorithm

The nearest-neighbor classification algorithm is widely used in machine learning. “Define by their characteristic of classifying unlabeled examples by assigning them the class of similar labeled examples.”

Problems like facial recognition used in Iphone X and movie recommendations used by Neflix can all be solved by using K-NN algorithms. Machine learning is already in our daily life.）

It would be useful if we could apply a real world problem by apply K-NN algorithm.

The importance of choosing the right criteria to analyze. The illustration of the Tomato example tells us the process of designing such project. But I wonder whether a Tomato is a fruit or vegetable should be a biological problem but not judged by only two categories : sweetness and crunchiness. This lead to another question I have: how to make sure the categories we investigate is the key category to set the category apart.

The importance of labeling data correctly

The key to build a successful k-NN algorithm depends on the precision of labeling data. This is extremely crucial in image analysis. Most of image database are not labeled. We have to rely on manually label all images.

Labeling is a practical problem in medical image analysis. Because of HIPPA regulation, the use of patients’ image analysis are highly regulated. This create another problem, only doctors can label medical images correctly and it may contain errors too.

Manually labeling images may slow the process and limit the number of the database. The study did by Google for eye problems among diabetic patients used only 135,000 images. In terms of database, it is not a big number. It may affect the correctness of the final model.

It may be great to test building a model use a small number of images.
